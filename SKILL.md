# Brand PPT / Keynote Skill

이 스킬은 사용자가 제공하는 **디자인 시스템**과 **브랜드 에셋**을 기반으로 일관된 스타일의 프레젠테이션을 생성합니다.

---

## 입력값 체크리스트

사용자에게 다음 항목을 받았는지 확인하세요. 없는 항목은 요청하세요.

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| 디자인 시스템 `.md` 파일 | 필수 | 레이아웃 규칙, 컴포넌트 사용법, 간격 등 |
| 레퍼런스 이미지 | 권장 | 슬라이드 스타일 레퍼런스 (스크린샷, 목업 등) |
| 주요 컬러 | 필수 | 브랜드 색상 (HEX 코드 또는 색상명) |
| 폰트 정보 | 필수 | 헤더/바디 폰트명 |
| 콘텐츠/스토리 | 필수 | PPT에 담을 내용 (자유로운 형식으로 OK) |
| 출력 형식 | 필수 | `.pptx` 또는 Keynote (`.key`) |
| 청중 | 권장 | 발표 대상 (투자자, 고객, 내부 팀, 임원 등) |
| 발표 시간 | 권장 | 전체 발표 시간 (슬라이드 수 산정 기준: 1장 ≈ 1~2분) |
| 슬라이드 수 | 권장 | 목표 또는 제한 페이지 수 |
| 발표 언어 | 권장 | 한국어 / 영어 / 기타 (폰트 선택에 영향) |

---

## 발표 컨텍스트 분석

입력값을 바탕으로 아래 항목을 결정한 뒤 슬라이드 제작에 반영합니다.

### 청중 분석 → 톤 & 깊이 결정
| 청중 유형 | 톤 | 전문 용어 | 강조 포인트 |
|-----------|-----|-----------|------------|
| 투자자 | 간결·자신감 | 최소화 | 숫자, 시장 기회, ROI |
| 고객 | 친근·설득 | 쉬운 언어 | 문제 해결, 혜택 |
| 내부 팀 | 실무·직접적 | 내부 용어 OK | 실행 계획, 역할 |
| 임원진 | 전략·요약 | 핵심만 | 의사결정 포인트 |

### 발표 시간 → 슬라이드 수 산정
- 정보 미제공 시 기본값: **1장 = 1.5분** 기준으로 역산
- 예: 10분 발표 → 약 6~7장 (표지/마무리 포함)
- 문서 공유용(발표자 없음)은 슬라이드 수를 1.5~2배로 늘리고 텍스트 밀도 높임

### 발표 형식 → 텍스트 밀도 결정
- **발표용** (발표자가 직접 설명): 키워드·이미지 중심, 텍스트 최소화
- **문서 공유용** (자료 단독으로 읽힘): 문장형 설명, 맥락 충분히 포함
- **온라인/화상**: 폰트 최소 24pt, 대비 강하게, 복잡한 레이아웃 지양
- **오프라인/현장**: 폰트 최소 20pt, 뒷자리 가독성 고려

## 워크플로우

### 1단계: 에셋 분석

**디자인 시스템 md 읽기**
- 레이아웃 그리드, 여백, 컴포넌트 규칙 파악
- 타이포그래피 스케일 (헤더/서브/바디 사이즈)
- 색상 사용 규칙 (Primary / Secondary / Accent / Background)

**레퍼런스 이미지 분석**
- 슬라이드 구조 패턴 파악 (텍스트 배치, 이미지 위치)
- 반복되는 디자인 모티프 추출 (아이콘 처리, 구분선, 카드 등)
- 전체적인 밀도와 여백 스타일 파악

### 2단계: 콘텐츠 → 슬라이드 구조화

발표 컨텍스트(청중·시간·형식)를 반영해 슬라이드 단위로 변환합니다.

```
스토리 → 슬라이드 구조 예시:
- 도입부 → 타이틀 슬라이드
- 문제 정의 → 1-2 슬라이드 (텍스트 + 시각 요소)
- 핵심 메시지 → 주요 콘텐츠 슬라이드들
- 데이터/근거 → 차트/통계 슬라이드
- 결론/CTA → 마무리 슬라이드
```

슬라이드 개수는 **발표 시간 기준**으로 산정하되, 각 슬라이드는 **하나의 메시지**에 집중.

### 3단계: 파일 생성

#### PPTX 생성 시
`/mnt/skills/public/pptx/SKILL.md`를 읽고 pptxgenjs 방식으로 생성.

핵심 규칙:
- 디자인 시스템의 컬러/폰트를 코드에 상수로 선언해서 사용
- 레퍼런스 이미지의 레이아웃 패턴을 최대한 재현
- 슬라이드마다 레이아웃을 변주 (단조로움 방지)
- 청중 유형에 맞는 텍스트 밀도와 톤 적용
- 비주얼 QA 필수 (pptx SKILL.md의 QA 섹션 참조)

#### Keynote (.key) 생성 시
Keynote는 바이너리 포맷이라 직접 생성 불가 → **대안 접근법** 사용:

**옵션 A (권장)**: PPTX를 먼저 만들고 변환 안내
```
1. 동일한 내용을 .pptx로 생성
2. "Keynote에서 파일 > 열기로 pptx를 불러오시면 Keynote로 변환됩니다" 안내
3. 주의사항: 폰트, 특수 애니메이션은 Keynote에서 재확인 필요
```

**옵션 B**: HTML 슬라이드로 생성 후 안내
```
1. reveal.js 또는 순수 HTML/CSS로 브라우저 기반 슬라이드 생성
2. "Keynote 네이티브 파일이 필요하시면 PPTX 버전을 Keynote에서 열어주세요" 안내
```

사용자에게 어떤 옵션을 원하는지 확인 후 진행.

### 4단계: 디자인 검증

디자인 시스템과 결과물을 비교 체크:

- [ ] 컬러가 디자인 시스템 팔레트 내에서만 사용됨
- [ ] 폰트가 지정된 폰트만 사용됨
- [ ] 레이아웃 여백이 디자인 시스템의 그리드/스페이싱 기준 충족
- [ ] 레퍼런스 이미지의 스타일 무드와 일치
- [ ] 텍스트 오버플로우 없음
- [ ] 슬라이드간 디자인 일관성 유지
- [ ] 청중 유형에 맞는 텍스트 밀도 적용됨
- [ ] 발표 시간 대비 슬라이드 수가 적절함
- [ ] 발표 언어에 맞는 폰트 사용됨

## 디자인 원칙 (브랜드 스타일 적용 시)

> 기본 pptx 스킬의 디자인 가이드라인보다 **디자인 시스템 md 파일이 항상 우선**합니다.
> 충돌 시 디자인 시스템을 따르세요.

- 사용자가 준 컬러/폰트를 변경하지 말 것
- 레퍼런스 이미지에 없는 디자인 요소를 임의로 추가하지 말 것
- "AI스러운" 기본 스타일 (크림 배경, 파란 강조색, 타이틀 하단 구분선) 절대 사용 금지

## 사용자에게 물어볼 것 (모호한 경우)

**발표 컨텍스트**
- 청중이 누구인가? (투자자 / 고객 / 내부 팀 / 임원진 / 기타)
- 청중의 해당 주제 사전 지식 수준은? (전문가 / 일반인)
- 발표 총 시간은? (슬라이드 수 산정에 사용)
- 발표자가 직접 설명하는 형식인가, 자료 단독으로 배포되는 형식인가?
- 온라인(화상) 발표인가, 오프라인(현장) 발표인가?
- 발표 언어는? (한국어 / 영어 / 기타)

**분량 & 형식**
- 슬라이드 수 제한 또는 목표치가 있는가?
- 슬라이드 비율: 16:9 (기본) vs 4:3?
- 애니메이션/전환 효과 필요 여부?
